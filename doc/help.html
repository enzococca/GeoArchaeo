GeoArchaeo - Analisi Geostatistica per l'Archeologia

GeoArchaeo è un plugin QGIS per l'analisi geostatistica avanzata di dati archeologici, con supporto per GPR, magnetometria, analisi del terreno e pattern recognition.

Funzionalità Principali

1. Analisi Variogramma

Calcola la struttura spaziale dei tuoi dati archeologici.

Cosa fa: Analizza come i valori cambiano con la distanza
Risultato: Grafico che mostra la correlazione spaziale
Uso: Carica un layer di punti → Seleziona il campo da analizzare → Esegui

2. Kriging (Interpolazione)

Crea mappe continue dai tuoi punti di campionamento.

Cosa fa: Interpola valori tra i punti misurati
Risultato: Raster con valori stimati su tutta l'area
Parametri:
  • Nugget: Variabilità a distanza zero (errore di misura)
  • Sill: Varianza massima raggiunta
  • Range: Distanza di correlazione

3. Machine Learning

Identifica pattern e anomalie nei dati archeologici.

Metodi disponibili:
  • K-Means: Raggruppa siti simili
  • DBSCAN: Trova cluster di forma irregolare
  • Random Forest: Classifica siti (richiede training)
  • Isolation Forest: Identifica anomalie

4. Campionamento Ottimale

Suggerisce dove effettuare nuovi campionamenti.

Cosa fa: Calcola posizioni ottimali per nuove misure
Risultato: Layer con punti suggeriti (stelle)
Metodi: Maximin, Riduzione varianza, Stratificato

Workflow Tipico

  1. Carica i dati: Usa "Carica Layer di Test" per esempi
  2. Esplora i pattern: Usa il variogramma per capire la struttura spaziale
  3. Interpola: Crea raster con Kriging
  4. Analizza: Usa ML per trovare pattern o anomalie
  5. Pianifica: Usa il campionamento ottimale per nuove misure

Tips

  • Per dataset grandi (>1000 punti), usa il Batch Kriging dal Processing
  • Il Co-Kriging è utile quando hai GPR + Magnetometria
  • L'analisi composizionale è ideale per dati del terreno (sabbia/limo/argilla)
  • Salva sempre i parametri del variogramma per riutilizzarli

Risoluzione Problemi

Il raster non appare? Verifica che ci siano abbastanza punti nell'area
Errore nel kriging? Controlla che il campo selezionato contenga valori numerici
ML non funziona? Assicurati di avere almeno 2 layer caricati
